# 0.5.5 — 2026-05-23

## Bug Fix

- **`hono` added as direct dependency (`^4.12.22`)** — the lockfile resolved `hono` to 4.12.19 at the top level while `mcp-ts-core` needed `^4.12.22`, causing bun to install a second copy nested inside `mcp-ts-core/node_modules/`. Two Hono instances mean two distinct `GET_MATCH_RESULT` symbols; `@hono/otel` imported `matchedRoutes` from the top-level copy but routing ran through the nested copy, so every HTTP request threw `TypeError: undefined is not an object (evaluating 'c.req[GET_MATCH_RESULT][0]')` and returned 500. Adding `hono` as a direct dependency forces a single hoisted copy, eliminating the symbol mismatch.
